Introduction to the Swift Programming Language
Key Language Concepts in Swift
  • first-class functions
  • lambda expressions
  • higher-order functions (map, filter, sorted, reduce)
  • first-class closures
  • partial function application
  • continuation passing style
  • pattern matching
  • strong typing
  • type systems
  • lazy evaluation
  • list comprehension
  • type inference
  • dynamic dispatch
Core Swift
  • official language for apple development
  • created for safety and software that runs fast
  • cleaner syntax than objective-c
  • primitive types: integer, string, float, double, boolean, character
  • three collection types: arrays, sets, dictionaries
  • memory is automatically managed
  • null safety
  • tuples and multiple returns
  • generics
  • structs
  • automatic json encoding and decoding
  • automatic reference counting
  • open source (community support and third-party support)
  • designed for safety
  • designed for cross-platform compatibility
Functions
  • you can type functions
    • func greet(greeting: String) -> String {
    \tlet g = greeting + " Joe!"
    \treturn g
    }
  • you can return multiple values
    • function firstTwo(array: [Int]) -> (first: Int, second: Int) {
    \treturn (array[0], array[1])
    }
  • you can input a variable number of parameters using the ``..."
    • func avg(_ numbers: Double...) -> Double {
    \tvar total: Double = 0
    \tfor number in numbers {
    \t\ttotal += number
    \t}
    \treturn total / Double(numbers.count)
    }

    avg(1)
    avg(1, 2, 3, 4)
    avg(1, 2, 3, 5.4, 3.2, 0.3)
Parameters
  • you must label your parameters, by default, in functions
    • func greet(greeting: String) -> String {
    \tlet g = greeting + " Joe!"
    \treturn g
    }
    In this example, "greeting" is the parameter label. You can now reference the parameter in the function using the label.
  • using a ``_" will make it so you don't have to label you parameter
    • func greet(_ greeting: String) -> String {
    \tlet g = greeting + " Joe!"
    \treturn g
    }
  • you can use the ``inout" keyword to make a variable call-by-reference
    • func greet(_ greeting: inout String) -> String {
    \tlet g = greeting + " Joe!"
    \treturn g
    }
Typing
  • swift is a ``type-safe'' language
  • type inference
    let pi = 3.14159
    // pi is inferred to be of type Double

    let one = 1
    // one is inferred to be of type Int
  • strong typing
    let one: Int = 1
    // one is typed to Int

    let two: Int = 1.27272
    // this will give an error since two in typed to an Int and the value assigned is a Double
  • type aliases
    typealias IntegerType = Int
    // IntegerType is aliased as an Int
Tuples
  • like a struct. contains heterogeneous types
    • let value = (20, "Hi")
    // value is inferred to be of the type (Int, String)
  • decomposition
    • let (age, greeting) = value
    print("His age is \(age)")
    // prints "His age is 20"
Collections
  • three types of collections: arrays, sets, and dictionaries
  • assigning a collection to a variable makes it mutable
  • assigning a collection to a constant makes it immutable
  • arrays
    • stores values of the same type in an ordered fashion
    • can create an array with an empty value
      var arrayOfInts = [Int]()
      or with a default value
      var arrayOfInts = Array(repeating: 0.0, count: 3)
      or by adding two array together
      var arrayOfIntsOne = Array(repeating: 0.0, count: 3)
      var arrayOfIntsTwo = Array(repeating: 1.0, count: 5)
      var addedArrays = arrayOfIntsOne + arrayOfIntsTwo
    • Other methods you can use with arrays
      • isEmpty
      • append
      • insert
      • remove
      • removeLast
  • sets
    • stores values of the same type in an unordered fashion
    • values in a set get a hash value
    • set operators
      • intersection
      • symmetricDifference
      • union
      • subtracting
      • isSubset
      • isSuperset
      • isStrictSubset
      • isDisjoint
  • dictionaries
    • stores associations between keys of the same type and values of the same type with no ordering
First Class Functions
  • functions in swift are first class. You can assign them to a variabl, return them from a function, and pass them into a function.
    • func add(_ a: Int, _ b: Int) -> Int {
    \treturn a + b
    }

    // assignment
    let t: (Int, Int) -> Int = add
    t(1, 2)

    // parameter passing
    func printAdd(_ f: (Int, Int) -> Int, _ a: Int, _ b: Int) {
    \tprint("Result: \(f(a, b))")
    }
    printAdd(add, 1, 2)

    // returning
    func sub(_ a: Int, _ b: Int) -> Int {
    \treturn b - a
    }

    func returnSub() -> (Int, Int) -> Int {
    \treturn sub
    }

    returnSub()(1, 2)

Closures
  • you can define an use closures
    • func makeIncrementer(forIncrement amount: Int) -> () -> Int {
    \tvar runningTotal = 0
    \tfunc incrementer() -> Int {
    \t\trunningTotal += amount
    \t\treturn runningTotal
    \t}
    \treturn incrementer
    }

    let incTen = makeIncrementer(forIncrement: 10)
    let incFour = makeIncrementer(forIncrement: 4)

    incTen()
    incTen()
    incFour()
    incTen()
    incFour()
  • you can also use autoclosures
    • var customers = ["Ajay", "Tom", "Tyler"]
    let serve = { customers.remove(at: 0) }
Null Safety
  • values in swift cannot be nil by default
  • if you want a value to be nil, you use optionals
  • optionals represent two possibilities: the value is present, or the value is not present
  • use ? to allow a value to be nil
    • var one: Int? = 1
    // one contains an actual Int value of 1

    one = nil
    // one now contains no value
Lazy Properties
  • lazy properties are variables that are not create until they are used
    • class Importer {
    \tvar filename = "test.txt"
    }

    class Manager {
    \tlazy var importer = Importer()
    \tvar file = "a"
    }

    let manager = Manager()
    manager.file
    manager.importer // lazy properties are not created until they are first used
Computed Properties
  • computed properties are variables that are created automatically
    • struct Square {
    \tvar height: Int
    \tvar width: Int
    \tvar area: Int {
    \t\tget {
    \t\t\treturn height * width
    \t\t}
    \t}
    }

    var u = Square(height: 5, width: 10)
    u.area
Initilization and Deinitialization
  • initialization is code that will run when the object is created
  • deinitialization is code that will run when the object is destroyed
    • class Bank {
    \tstatic var coinsInBank = 10000
    \tstatic func send(coins numCoins: Int) -> Int {
    \t\tcoinsInBank -= numCoins
    \t\treturn numCoins
    \t}
    \tstatic func receive(coins: Int) {
    \t\tcoinsInBank += coins
    \t}
    }

    class Person {
    \tvar coinsInPocket: Int
    \tinit() {
    \t\tcoinsInPocket = Bank.send(coins: 100)
    \t}
    \tfunc getCoins(amount: Int) {
    \t\tcoinsInPocket += Bank.send(coins: amount)
    \t}
    \tdeinit {
    \t\tBank.receive(coins: coinsInPocket)
    \t}
    }

    var v: Person? = Person()
    v!.getCoins(amount: 200)
    v!.coinsInPocket
    Bank.coinsInBank
    v = nil
    Bank.coinsInBank
Extensions
  • extensions allow you to add variables or functions to a type
    • // adds functionality to an already existing thing
    // converts Double values to meters based on what unit they are
    extension Double {
    \tvar km: Double { return self * 1_000.0 }
    \tvar m: Double { return self }
    \tvar cm: Double { return self / 100.0 }
    \tvar mm: Double { return self / 1_000.0 }
    v\tar ft: Double { return self / 3.28084 }
    }
    let oneInch = 25.4.mm
    print("One inch is \(oneInch) meters")

    let threeFeet = 3.ft
    print("Three feet is \(threeFeet) meters")

    // adds functionality to all ints where you can addTwo
    extension Int {
    \tmutating func addTwo() {
    \t\tself = self + 2
    \t}
    }

    var y = 3
    y.addTwo()

    // adds functionality to all ints where it will print out "hi" the number the value is
    extension Int {
    \tfunc repetitions() {
    \t\tfor _ in 0..<self {
    \t\t\tprint("Hi!")
    \t\t}
    \t}
    }

    6.repetitions()
Protocols
  • protocols are blueprints for the struct
    • protocol Name {
    var name: String { get}
    }

    struct Player: Name {
    \tvar first: String
    \tvar last: String

    \tvar name: String {
    \t\treturn first + " " + last
    \t}
    }

    let z = Player(first: "Ajay", last: "Patnaik")
    z.name
Generics
  • you can use generic typing in swift
    • func genericSwap(_ a: inout T, _ b: inout T) {
    \tlet temp = a
    \ta = b
    \tb = temp
    }

    var aa = 3
    var bb = 4
    genericSwap(&aa, &bb)

    var cc = "Hi"
    var dd = "Bye"
    genericSwap(&cc, &dd)
Custom Operators
  • you can create custom prefix, infix, or postfix operators
    • postfix operator +++
    extension Int {
    \tstatic postfix func +++ (num: inout Int) -> Int {
    \t\tnum = num + num + num
    \t\treturn num
    \t}
    }

    var ee: Int = 3
    let ff = ee+++
Partial Function Application
  • using first class functions, you can create partial function application
    • func partialAdd(a: Int) -> (Int) -> Int {
    \treturn { b in a + b }
    }

    let addThree = partialAdd(a: 3)
    let partialSum = addThree(10)
Higher-order Functions
  • you can use higher-order functions in swift
    • let gg = ["Ajay", "Tom", "Tyler"]
    // $0.lowercased could be any function
    let lowercase = gg.map { $0.lowercased() }
List Comprehension
  • you can do list comprehension in swift
    • let evenSquared = (0..<10).filter({ $0 % 2 == 0 }).map({ $0 * $0 })
JSON encoding and decoding
  • helpful for using API data
    • struct Student {
    \tvar name: String
    \tvar age: Int

    \tinit(_ name: String, _ age: Int) {
    \t\tself.name = name
    \t\tself.age = age
    \t}
    }

    var student = Student("Ajay", 21)

    extension Student: Codable, Equatable {}
    // Codable is a swift library that allows you to encode and decode // Equatable is a swift library that allows you to better use == and != with types
    import Foundation // framework that gives access to data types, collections, and operating-system services
    let encoder = JSONEncoder()
    try encoder.encode(student) // encodes the student object into a JSON format
Creating an app in swift
  • UIKit is a default package that allows for the construction of graphical and event-driven user interface
  • Apple only allows for the creation of apps on xcode, which is only available on apple products
  • six files and 1 folder are created when a swift project is created
    • AppDelegate.swift
      • this file creates the Application and serves as the main entry point for the app
    • ViewController.swift
      • this is where you modify the elements and the element's events in the GUI (such as buttons presses and images)
    • Main.storyboard
      • this is a drag-and-drop like page when you open it in xcode. this is the main screen for the app and you can drag any UI elements onto this page
    • Asses.xcassets
      • this is a folder to put your assets (such as images and videos)
    • LaunchScreen.storyboard
      • this is also drag-and-drop like page when you open it in xcode. this is the screen that shows up when your app is loading
    • Info.plist
      • this file is used to list all of the apple configs for the app (such as app name and app identifier)
